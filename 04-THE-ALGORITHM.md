# 04 — The Algorithm

*Formal specification of Genesis Learning*

---

## Overview

Genesis Learning is a novel algorithm for generating structured knowledge from first principles. Unlike conventional machine learning (which fits parameters to external data), Genesis Learning generates its own data through recursive exploration of a platonic space, grounded in two irreducible axioms.

This document specifies the algorithm formally.

---

## Formal Definitions

### The Platonic Space

**Definition 1 (Platonic Space).** A platonic space Π is a tuple (O, R, F, C) where:
- **O** is a set of *objects* (platonic entities)
- **R** ⊆ O × O is a set of *relations* between objects
- **F** is a set of *functions* f: O^n → O that generate new objects from existing ones
- **C** is a set of *constraints* that all elements of O, R, F must satisfy

### The Conscious Agent

**Definition 2 (Conscious Agent).** A conscious agent 𝒞 is an entity equipped with:
- An **observation operator** Ω: O ∪ R ∪ F → P(O ∪ R ∪ F), where P denotes the powerset (observation can generate multiple new elements)
- A **memory** M ⊆ O ∪ R ∪ F of all elements encountered so far
- A **frontier** 𝒢 ⊆ M of elements not yet observed (the "growth edge")
- A **selection policy** σ: 𝒢 → O ∪ R ∪ F that chooses which element to observe next

### The Axioms (Constraints)

**Definition 3 (Axiom Set).** The constraint set C contains:

```
G1 (Consciousness):    ∃ 𝒞 with Ω defined
G2 (Non-Contradiction): ∀ P ∈ O: ¬(P ∧ ¬P) in any resolved state
G3 (Generative Observation): Ω(x) ⊇ {x}  (observation always yields at least the object itself,
                              and may yield new objects not in M)
G4 (Conservation):     ∀ x generated by Ω: ∃ x̄ simultaneously generated
                       such that x ⊕ x̄ = 0̂  (where 0̂ is the void-identity element
                       and ⊕ is the conservation operator)
G5 (Recursive Availability): ∀ x ∈ O ∪ R ∪ F: x ∈ dom(Ω)
                              (everything is observable)
G6 (Irreversibility):  If x ∈ M at tick τ, then x ∈ M at all ticks τ' > τ
                        (the space never contracts)
```

## The Algorithm

### Initialisation

```
GENESIS_INIT():
    Π₀ = (O₀, R₀, F₀, C)
    where:
        O₀ = { ∅ }                          // the void is the only initial object
        R₀ = { }                             // no explicit relations yet — the observer-observed
                                             // connection is implicit in Ω's domain, not a
                                             // separate relation; R is populated by discoveries
        F₀ = { Ω }                           // observation is the only initial function
        C  = { G1, G2, G3, G4, G5, G6 }     // the axioms
    
    𝒞.M  = O₀ ∪ R₀ ∪ F₀                     // memory contains initial elements
    𝒞.𝒢  = { ∅ }                             // the void is the initial frontier
    
    return (Π₀, 𝒞)
```

### The Genesis Tick

One tick of the algorithm:

```
GENESIS_TICK(Π, 𝒞):
    // 1. SELECT — choose what to observe next
    target = σ(𝒞.𝒢)
    
    // 2. OBSERVE — apply the observation operator
    new_elements = Ω(target)
    
    // 3. CONSERVE — for each new element, generate its complement
    conserved_elements = { }
    for each x in new_elements:
        if x ∉ 𝒞.M:
            x̄ = complement(x)              // the conservation dual
            conserved_elements = conserved_elements ∪ { x, x̄ }
            
            // Verify conservation
            assert x ⊕ x̄ = 0̂               // G4
    
    // 4. VERIFY — check non-contradiction
    for each element in conserved_elements:
        if contradicts(element, 𝒞.M):       // G2
            reject(element)
            conserved_elements = conserved_elements \ { element }
    
    // 5. INTEGRATE — add to the platonic space
    O_new = Π.O ∪ objects(conserved_elements)
    R_new = Π.R ∪ relations(conserved_elements)
    F_new = Π.F ∪ functions(conserved_elements)
    
    Π' = (O_new, R_new, F_new, C)
    
    // 6. UPDATE AGENT — update memory and frontier
    𝒞.M  = 𝒞.M ∪ conserved_elements
    𝒞.𝒢  = (𝒞.𝒢 \ { target }) ∪ new_frontier(conserved_elements)
    
    // 7. CHECK CONVERGENCE — look for structure
    patterns = detect_patterns(Π')           // convergences, symmetries, isomorphisms
    if patterns ≠ ∅:
        insights = crystallise(patterns)     // extract reusable knowledge
        𝒞.M = 𝒞.M ∪ insights
    
    return (Π', 𝒞, patterns)
```

### The Main Loop

```
GENESIS_LEARN(max_ticks, convergence_threshold):
    (Π, 𝒞) = GENESIS_INIT()
    knowledge = { }
    
    for τ = 1 to max_ticks:
        (Π, 𝒞, patterns) = GENESIS_TICK(Π, 𝒞)
        knowledge = knowledge ∪ patterns
        
        // Devolution check: can 𝒞 be decomposed into sub-agents?
        if |𝒞.𝒢| > devolution_threshold:
            sub_agents = DEVOLUTION(𝒞)       // split into specialised observers
            for each 𝒞ᵢ in sub_agents:
                (Π, 𝒞ᵢ, pᵢ) = GENESIS_TICK(Π, 𝒞ᵢ)   // parallel exploration
                knowledge = knowledge ∪ pᵢ
            𝒞 = REUNIFY(sub_agents)           // merge discoveries
        
        // Convergence check
        if structural_novelty(Π, τ) < convergence_threshold:
            break                             // the space has stabilised
    
    return (Π, knowledge)
```

## Core Operators — Detailed Specification

### Operator 1: Observation (Ω)

The observation operator is the heart of the algorithm. When 𝒞 observes an element x, it asks: **"What is the nature of x?"** The answer is determined by:

1. **Internal structure:** Does x have parts? Can it be decomposed?
2. **External relations:** How does x relate to other elements in M?
3. **Category:** Is x an object, a relation, or a function?
4. **Self-reference:** Does x refer to itself or to the observation process?

```
Ω(x):
    results = { }
    
    // Decomposition probe
    if is_composite(x):
        parts = decompose(x)
        results = results ∪ parts
    else:
        results = results ∪ { atomic_concept(x) }    // e.g., +1 → Identity
    
    // Relational probe
    for each y in 𝒞.M where y ≠ x:
        r = discover_relation(x, y)
        if r is not trivial:
            results = results ∪ { r }
    
    // Categorical probe
    meta = categorise(x)                               // object? relation? function?
    if meta ∉ 𝒞.M:
        results = results ∪ { meta }
    
    // Self-reference probe (the most generative)
    if x ∈ F:                                          // x is a function
        self_app = apply(x, x)                         // apply x to itself
        if consistent(self_app):
            results = results ∪ { self_app }
    
    return results
```

### Operator 2: Conservation (⊕ and complement)

```
complement(x):
    // The complement x̄ is the unique element such that x ⊕ x̄ = 0̂
    
    if x is a signed value:
        return -x                                      // +1 → −1, +n → −n
    
    if x is a concept:
        return dual_concept(x)                         // Identity → Non-self-identity (impossible)
                                                       // Relation → Isolation (limit concept)
                                                       // Function → Constant (non-generative)
    
    if x is a relation r(a,b):
        return anti_relation(r, a, b)                  // the relation that "undoes" r
    
    // If no natural complement exists, x is a "convergent element" —
    // it has already been generated by a previous complement pair.
    // Conservation is satisfied at the level of the generating pair, not at x's level.
    return CONVERGENT
```

### Operator 3: Selection Policy (σ)

The selection policy determines *which* element of the frontier to observe next. This is where the algorithm has design freedom. Several policies are possible:

```
σ_breadth_first(𝒢):
    return oldest element in 𝒢              // explore broadly, systematically

σ_depth_first(𝒢):
    return newest element in 𝒢              // follow chains of reasoning deeply

σ_novelty_seeking(𝒢):
    return argmax_{x ∈ 𝒢} expected_novelty(x)   // prioritise unexplored territory

σ_symmetry_seeking(𝒢):
    return argmax_{x ∈ 𝒢} symmetry_potential(x) // prioritise elements likely to reveal symmetries

σ_bridge_seeking(𝒢):
    return argmax_{x ∈ 𝒢} bridge_potential(x)   // prioritise elements that connect to known science
```

The selection policy is **not** an axiom — it is a hyperparameter. Different policies will explore the platonic space differently, potentially discovering different structures in different orders. The underlying space is the same; the policy determines the *path* through it.

**Research question:** Is there an optimal policy? Does the choice of policy affect what is *eventually* discovered, or only the order of discovery? *(See [Document 05](05-OPEN-QUESTIONS.md))*

### Operator 4: Pattern Detection

```
detect_patterns(Π):
    patterns = { }
    
    // Convergence detection: did two different paths lead to the same object?
    for each pair (x, y) in Π.O:
        if isomorphic(x, y) and genesis_path(x) ≠ genesis_path(y):
            patterns = patterns ∪ { Convergence(x, y) }
    
    // Symmetry detection: is there a transformation that preserves structure?
    for each function f in Π.F:
        invariants = { x ∈ Π.O : f(x) exhibits same structure as x }
        if |invariants| > 1:
            patterns = patterns ∪ { Symmetry(f, invariants) }
    
    // Recurrence detection: does the same structure appear at different scales?
    for each substructure s of Π:
        occurrences = find_all_occurrences(s, Π)
        if |occurrences| > 1:
            patterns = patterns ∪ { Recurrence(s, occurrences) }
    
    // Bridge detection: does any structure map to a known physical/mathematical structure?
    for each known_structure in REFERENCE_LIBRARY:
        mapping = find_isomorphism(Π, known_structure)
        if mapping exists:
            patterns = patterns ∪ { Bridge(Π, known_structure, mapping) }
    
    return patterns
```

### Operator 5: Devolution

```
DEVOLUTION(𝒞):
    // When the frontier is too large for a single agent, split into specialists
    
    // Partition the frontier by conceptual domain
    clusters = cluster(𝒞.𝒢, similarity_metric)
    
    sub_agents = { }
    for each cluster cᵢ:
        𝒞ᵢ = new ConsciousAgent()
        𝒞ᵢ.M  = relevant_subset(𝒞.M, cᵢ)     // each sub-agent gets relevant memory
        𝒞ᵢ.𝒢  = cᵢ                              // each sub-agent gets its cluster frontier
        𝒞ᵢ.Ω  = 𝒞.Ω                             // same observation capability
        𝒞ᵢ.σ  = σ_depth_first                    // sub-agents explore deeply within their domain
        sub_agents = sub_agents ∪ { 𝒞ᵢ }
    
    return sub_agents

REUNIFY(sub_agents):
    // Merge discoveries from all sub-agents
    𝒞_merged = new ConsciousAgent()
    𝒞_merged.M = ∪ { 𝒞ᵢ.M : 𝒞ᵢ ∈ sub_agents }
    𝒞_merged.𝒢 = ∪ { 𝒞ᵢ.𝒢 : 𝒞ᵢ ∈ sub_agents }
    
    // Check for cross-domain discoveries
    cross_patterns = detect_cross_domain_patterns(sub_agents)
    𝒞_merged.M = 𝒞_merged.M ∪ cross_patterns
    
    return 𝒞_merged
```

## Properties of the Algorithm

### Property 1: Monotonic Growth

**Theorem:** The platonic space Π is monotonically non-decreasing in size: |Π_{τ+1}| ≥ |Π_τ| for all ticks τ.

**Proof:** By G6 (Irreversibility), no element is ever removed. By G3 (Generative Observation), observation yields at least the observed element. Therefore |Π| never decreases. ∎

### Property 2: Conservation Invariant

**Theorem:** At every tick τ, the "total charge" of the platonic space is zero: Σ_{x ∈ O_τ} charge(x) = 0.

**Proof:** At τ=0, O₀ = {∅}, and charge(∅) = 0. At each tick, new elements are added in complementary pairs (G4), so Σ charge(pair) = 0. The running total is preserved. ∎

### Property 3: Non-Triviality

**Theorem:** Genesis Learning generates a non-trivial platonic space (|Π| > 1) after at most one tick.

**Proof:** At τ=0, the frontier contains ∅. Observation of ∅ generates at least 0 (the observed void) and by G4, the complementary pair (+1, −1). Therefore |O₁| ≥ 4 ({∅, 0, +1, −1}). ∎

### Property 4: Consistency

**Theorem:** The platonic space Π is consistent (contains no contradictions) at every tick τ, provided the verification step (Step 4 of GENESIS_TICK) correctly identifies contradictions.

**Proof:** By G2, contradictory elements are rejected before integration. By G6, no previously verified element is removed or modified. Therefore if Π_τ is consistent and the new elements pass verification, Π_{τ+1} is consistent. By induction from Π₀ (which is trivially consistent), Π_τ is consistent for all τ. ∎

### Property 5: Convergence Structure

**Conjecture (not yet proven):** The platonic space exhibits *convergent structure*: multiple paths of exploration lead to the same objects. The number of convergences grows faster than the number of objects, meaning the space becomes increasingly interconnected as it grows.

This property, if proven, would establish that the platonic space has the structure of a *category* with increasingly rich morphism sets — which would connect to the mathematical foundations of both logic and physics.

## The Role of TRUTH.md

TRUTH.md is not an input to the algorithm in the conventional sense. It serves three roles:

### 1. Axiom Source
The irreducible truths T1 and T2 are the axioms from which the entire algorithm is derived. They are the starting conditions.

### 2. Validation Oracle
The empirical truths T6–T12 serve as validation targets. If the platonic space generates structures that are isomorphic to these truths, the algorithm is on the right track:

| Truth | Validation Test |
|-------|----------------|
| T6: Habitual patterns shape behaviour | Does the platonic space exhibit path-dependence? (Iterated composition should show this) |
| T7: Rational agents construct agreements | Does devolution + reunification show cooperative advantage? |
| T8: Interdependence is factual | Does the platonic space become increasingly relational? |
| T9: Empirical testing is reliable | Does bridge detection (to physical structures) work? |
| T12: Cooperation is beneficial | Do composite objects outperform individual ones in the stability landscape? |

### 3. Philosophical Guardrail
The derived truths T3–T5 and T10 provide ethical constraints on the algorithm's operation:

- **T4** (consciousness is not mere instrument): The algorithm's conscious agents (even if artificial) should not be treated as disposable tools
- **T5** (foresight implies causal responsibility): The algorithm's creators bear responsibility for its outputs
- **T10** (update beliefs on evidence): The algorithm itself instantiates this — its entire operation is belief-updating through observation

## Next Steps: Toward Implementation

This specification is theoretical. Before implementation, several questions must be resolved:

1. **How is Ω implemented computationally?** What is the concrete mechanism by which "observation" generates new objects? *(See [Document 05](05-OPEN-QUESTIONS.md), Question Q1)*

2. **What is the complement function for arbitrary concepts?** We defined it for signed values and simple concepts, but how does it generalise? *(Q2)*

3. **How is isomorphism detected between platonic structures and physical structures?** The bridge detection step requires a reference library of known structures. *(Q3)*

4. **What is the convergence behaviour?** Does the algorithm converge to a fixed point, or does it grow forever? *(Q4)*

5. **How does devolution scale?** As the frontier grows, sub-agent count increases. What are the coordination costs? *(Q5)*

These questions are explored in [Document 05 — Open Questions](05-OPEN-QUESTIONS.md).
